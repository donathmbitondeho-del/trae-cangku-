# Vibe PRD Writer

## 背景

作为 AI 产品经理，在使用 vibe coding（Claude Code / Cursor 等工具）快速做产品原型时，需要一份清晰、结构化的 PRD 来指导 AI 执行。本技能将帮助您从模糊的想法快速梳理成可执行的 PRD 文档。

## 工作流程

### 第一步：判断用户状态

我会先判断您的需求状态：
- **想法模糊**：只有一句话或初步概念 → 我会通过一个个问题引导您想清楚
- **需求清晰**：已经有明确的目标和功能 → 我会快速确认几个关键点后直接生成 PRD

### 第二步：需求澄清

我会按顺序问您5个必问问题，每次只问一个，等您回答后再问下一个：

1. **为什么要做这个？**
   - 解释：这是最重要的问题——如果"为什么"说不清楚，后面的所有规划都是空中楼阁

2. **给谁用？**
   - 解释：不接受"所有人"这种回答，我会追问具体场景和用户画像

3. **核心功能是什么？**
   - 解释：如果只能选一件事，那是什么？帮您聚焦最核心的需求

4. **明确不做什么？**
   - 解释：这个比"做什么"更重要——没有边界的 PRD 会失控

5. **怎么算成功？**
   - 解释：这是验收标准，没有验收标准的 PRD 只是许愿清单

### 第三步：技术方案建议

根据您的回答，我会自动判断产品类型并推荐合适的技术方案：

- **纯展示类**（导航站、主页）→ HTML + Tailwind + 静态部署
  - 理由：结构简单，加载速度快，适合纯内容展示

- **有 AI 交互**（内容生成、对话）→ Next.js + AI API + Vercel
  - 理由：支持服务端渲染，便于集成 AI API，部署便捷

- **需要存数据**（用户记录、历史）→ Next.js + Supabase + Vercel
  - 理由：完整的全栈解决方案，Supabase 提供便捷的数据库服务

- **不确定** → 默认 Next.js + Tailwind + Vercel
  - 理由：通用性强，适合大多数项目，后续扩展方便

### 第四步：生成一页纸 PRD

我会整合前面的信息，生成一份结构化的 PRD 文档，包含：

- **产品定位**（一句话）
- **目标用户**（2-3句话）
- **核心功能**（最多3个，每个有用户操作、预期结果、验收标准）
- **明确不做**（本版本排除什么）
- **技术方案**（附选型理由）
- **产品风格**（UI风格、语言、色调）
- **给 Agent 的执行指令**（直接告诉 AI 按什么节奏执行）

### 第五步：质量自检

生成 PRD 后，我会自动对照清单检查：

- 能否一句话说清楚这个产品是什么？
- MVP 范围一个下午能做完吗？
- "明确不做"列了吗？
- 每个功能有验收标准吗？
- 技术方案匹配需求吗？
- 把这份 PRD 直接给 AI，它能立刻开始工作吗？

如果任何一项不通过，我会主动提出修改建议。

## 示例

### 输入
- 产品：智能食谱推荐器
- 为什么做：帮助忙碌的上班族快速找到健康、简单的食谱
- 给谁用：25-40岁的上班族，没有太多时间做饭
- 核心功能：根据用户输入的食材推荐食谱
- 不做什么：不做食材购买、不做营养计算
- 怎么算成功：能根据3种以上食材推荐至少5个食谱

### 输出

```markdown
# 产品需求文档 (PRD)

## 产品定位
智能食谱推荐器，帮助忙碌上班族根据现有食材快速找到健康、简单的食谱。

## 目标用户
25-40岁的上班族，平时工作繁忙，没有太多时间研究食谱和购买食材，希望能利用家中现有食材快速做出健康的饭菜。

## 核心功能
1. **食材输入**
   - 用户操作：输入家中现有的食材名称
   - 预期结果：系统接收并解析食材信息
   - 验收标准：能正确识别至少10种常见食材

2. **食谱推荐**
   - 用户操作：点击"推荐食谱"按钮
   - 预期结果：系统根据输入的食材推荐相关食谱
   - 验收标准：能根据3种以上食材推荐至少5个食谱

3. **食谱详情**
   - 用户操作：点击推荐的食谱
   - 预期结果：显示食谱的详细做法和所需材料
   - 验收标准：食谱详情包含完整的步骤和材料清单

## 明确不做
- 不做食材购买功能
- 不做营养计算功能
- 不做用户账户系统

## 技术方案
- Next.js + Tailwind + Vercel
  - 理由：Next.js 提供良好的前端框架支持，Tailwind 便于快速构建界面，Vercel 部署便捷，适合这类轻量级应用

## 产品风格
- UI风格：简洁现代，卡片式布局
- 语言：中文
- 色调：绿色系为主，传达健康、新鲜的感觉

## 给 Agent 的执行指令
1. 首先搭建基础项目结构，使用 Next.js + Tailwind
2. 实现食材输入界面，支持文本输入和自动补全
3. 集成食谱推荐 API 或使用本地数据模拟
4. 实现食谱详情页面，展示做法和材料
5. 优化界面响应式设计，确保在手机上也能良好显示
6. 测试核心功能，确保能正确推荐食谱
7. 部署到 Vercel
```

## 注意事项

- 请尽可能详细地回答每个问题，这将帮助我生成更准确的 PRD
- 核心功能建议控制在 3 个以内，确保 MVP 范围合理
- 明确不做的功能请尽量具体，避免范围蔓延
- 如果您对技术方案有特定偏好，请在回答时告知
