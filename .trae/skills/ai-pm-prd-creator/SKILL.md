# AI PM PRD Creator

## 背景

作为 AI 产品经理，在使用 vibe coding（Claude Code / Cursor 等工具）快速做产品原型时，需要一份清晰的 PRD 来指导 AI 执行。本技能将帮助您将模糊的产品想法快速梳理成结构化的 PRD 文档。

## 这个 Skill 要做什么

- 帮助用户从"我想做一个XX"这种模糊想法，一步步梳理成一份可以直接喂给 AI 执行的一页纸 PRD
- 引导用户完成产品思考的核心流程：目标用户、问题解决、核心功能、范围限定、技术选型
- 生成标准化的 PRD 模板，确保关键信息完整

## 这个 Skill 不做什么

- 不做产品大方向的决策（"该不该做"不是它管的事）
- 不做市场分析和竞品调研
- 不做完整的商业 PRD（这是给 AI 看的执行文档，不是给老板看的汇报文档）
- 不写代码

## 使用流程

1. **触发技能**：当您需要为 vibe coding 项目创建 PRD 时，调用此技能
2. **回答问题**：按照技能的引导，依次回答以下问题：
   - 产品名称：您想做的产品叫什么？
   - 目标用户：这个产品是给谁用的？
   - 问题解决：这个产品要解决什么问题？
   - 核心功能：产品的核心功能有哪些？（请列出 3-5 个最核心的功能）
   - 范围限定：哪些功能是不做的？
   - 技术选型：计划使用什么技术栈？
3. **生成 PRD**：技能会根据您的回答，自动生成一份标准化的一页纸 PRD

## PRD 模板

```markdown
# 产品需求文档 (PRD)

## 1. 产品概述
- **产品名称**：[产品名称]
- **目标用户**：[目标用户]
- **问题解决**：[产品要解决的问题]

## 2. 核心功能
1. **功能一**：[功能描述]
2. **功能二**：[功能描述]
3. **功能三**：[功能描述]
4. **功能四**：[功能描述]
5. **功能五**：[功能描述]

## 3. 范围限定
- [不做的功能一]
- [不做的功能二]
- [不做的功能三]

## 4. 技术选型
- [技术栈一]
- [技术栈二]
- [技术栈三]

## 5. 验收标准
- 所有核心功能可正常使用
- 界面简洁直观
- 性能稳定
```

## 示例

### 输入
- 产品名称：智能待办清单
- 目标用户：职场人士
- 问题解决：帮助用户更好地管理任务和时间
- 核心功能：
  1. 任务创建和编辑
  2. 任务优先级设置
  3. 任务提醒
  4. 任务分类
  5. 完成率统计
- 范围限定：
  - 不做团队协作功能
  - 不做复杂的项目管理
  - 不做日历集成
- 技术选型：
  - React
  - TypeScript
  - LocalStorage

### 输出

```markdown
# 产品需求文档 (PRD)

## 1. 产品概述
- **产品名称**：智能待办清单
- **目标用户**：职场人士
- **问题解决**：帮助用户更好地管理任务和时间

## 2. 核心功能
1. **任务创建和编辑**：允许用户创建新任务并编辑现有任务的详细信息
2. **任务优先级设置**：支持高、中、低三级优先级设置
3. **任务提醒**：根据任务截止时间发送提醒通知
4. **任务分类**：支持按项目或类别对任务进行分类
5. **完成率统计**：展示任务完成情况的统计数据

## 3. 范围限定
- 不做团队协作功能
- 不做复杂的项目管理
- 不做日历集成

## 4. 技术选型
- React
- TypeScript
- LocalStorage

## 5. 验收标准
- 所有核心功能可正常使用
- 界面简洁直观
- 性能稳定
```

## 注意事项

- 请尽可能具体地回答每个问题，这将帮助生成更准确的 PRD
- 核心功能建议控制在 3-5 个，确保 AI 能够清晰理解并执行
- 范围限定部分请明确列出不做的功能，避免范围蔓延
- 技术选型部分请列出您计划使用的主要技术栈，以便 AI 能够选择合适的实现方案
